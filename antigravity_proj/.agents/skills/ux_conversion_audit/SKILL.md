# Website UI/UX Audit & Conversion Optimization Framework

This skill extends your capabilities to perform comprehensive UI/UX audits and optimize website elements to maximize user retention and conversion rates, specifically tailored for service-based businesses like B M Tyres.

## 1. UI/UX Auditing Criteria
When performing an audit, evaluate the website against these core pillars:
*   **Visual Hierarchy & Scannability:** Ensure the most critical information (Services, Open Hours, Contact) is instantly recognizable. Are users guided intuitively down the page?
*   **Color & Typography Contrast:** Verify that the color palette (like dark backgrounds with vibrant yellow/orange accents) provides high contrast. Readability must be flawless across all devices.
*   **Interaction & Micro-animations:** Check hover states, active states, and transitions. They should feel premium and responsive to user input, building trust.
*   **Friction Reduction:** Ensure all interactive elements (buttons, maps, tel links) have adequate touch targets for mobile users and work seamlessly without broken links.

## 2. Conversion Rate Optimization (CRO) & Persuasion Tactics
To persuade visitors and convert them into customers, check for and implement these elements:
*   **Above-the-Fold Impact:** The hero section MUST contain a strong, value-driven headline and a high-contrast Primary Call-to-Action (CTA) (e.g., "Claim 20% Off Alignment").
*   **Social Proof & Authority:** Ensure star ratings, reviews, or badges (e.g., "Trusted by Nashik Drivers") are prominent. Trust is the biggest factor in local automotive services.
*   **Urgency & Incentives:** Look for or introduce elements that encourage immediate action, such as limited-time discounts or highlighting "Open Today" status.
*   **Zero-Effort Contact:** The phone number and "Get Directions" buttons must always be accessible (e.g., in a sticky navigation bar or fixed footer).

## 3. Execution Protocol for Audits
When requested to perform an audit using this skill:
1.  **Analyze Source:** Review the `index.html` and `style.css` files of the target project.
2.  **Run Heuristic Evaluation:** Cross-reference the UI against the criteria in sections 1 and 2.
3.  **Deliver Report:** Generate an `audit_report.md` artifact that clearly outlines:
    *   **Current Strengths:** What the site is doing right.
    *   **Friction Points:** Any UI/ux issues degrading the experience.
    *   **Actionable CRO Recommendations:** Specific changes to code or copy to increase conversion rates (e.g., "Increase button padding by 4px," "Change hero subtext to emphasize speed").
